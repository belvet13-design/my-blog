# 오늘 배운 것

클로드 코드로 이 블로그를 만들면서, HTML과 CSS와 JavaScript가 각각 어떤 역할을 맡고 있는지 다시 한번 정리해봤습니다.

## HTML — 구조

`index.html`은 페이지의 뼈대만 담당합니다. 이 블로그에서는 헤더 하나와 콘텐츠가 들어갈 `<main id="app">` 컨테이너 하나뿐입니다. 실제 글 목록이나 본문은 전부 JavaScript가 채워 넣기 때문에, HTML 자체는 아주 단순하게 유지할 수 있었습니다.

## CSS — 표현

`style.css`는 색상, 여백, 폰트 크기, 반응형 레이아웃을 담당합니다. 이번에 인상 깊었던 부분은 **CSS 변수**로 라이트/다크 테마를 관리한 것입니다.

```
:root { --bg: #fff; --fg: #1a1a1a; }
[data-theme="dark"] { --bg: #121212; --fg: #eaeaea; }
```

색상을 하드코딩하지 않고 `var(--bg)`처럼 변수로만 참조하니, 테마 전체를 속성 하나로 뒤집을 수 있었습니다.

## JavaScript — 동작

JavaScript는 세 가지 일을 합니다.

1. **라우팅** — 주소창의 해시(`#/`, `#/post/slug`)를 읽어 어떤 화면을 보여줄지 결정합니다.
2. **데이터 가져오기** — `posts.json`과 마크다운 파일을 `fetch`로 불러옵니다.
3. **렌더링** — 가져온 마크다운을 직접 만든 파서로 HTML로 바꾸고, 화면에 그려 넣습니다.

## 정리

결국 세 언어의 역할은 이렇게 나뉩니다.

- HTML: 무엇이 있는가 (구조)
- CSS: 어떻게 보이는가 (표현)
- JavaScript: 어떻게 동작하는가 (동작)

프레임워크 없이 직접 만들어보니 이 세 가지 역할이 훨씬 선명하게 느껴졌습니다.
