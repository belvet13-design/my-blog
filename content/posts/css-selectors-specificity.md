# CSS 선택자와 우선순위 이해하기

오늘은 CSS를 쓰면서 항상 헷갈렸던 **선택자 우선순위(specificity)**를 정리해봤습니다.

## 선택자의 종류

CSS 선택자는 크게 세 가지로 나뉩니다.

- 태그 선택자: `p`, `div`
- 클래스/속성 선택자: `.post-title`, `[data-theme="dark"]`
- id 선택자: `#app`

같은 요소에 여러 규칙이 겹치면, 브라우저는 이 순서대로 **더 구체적인 선택자**를 우선시합니다.

## 우선순위 계산

우선순위는 대략 이런 식으로 점수가 매겨집니다.

1. id 선택자 1개 = 100점
2. 클래스/속성/가상클래스 선택자 1개 = 10점
3. 태그 선택자 1개 = 1점

예를 들어 `.post-list .post-item`은 클래스 2개라 20점이고, `#app p`는 id 1개 + 태그 1개라 101점입니다. 점수가 더 높은 규칙이 이깁니다.

```
#app p { color: blue; }      /* 101점 */
.post-list .post-item { color: red; } /* 20점 */
```

이 경우 두 규칙이 같은 요소를 가리키면 `color: blue`가 이깁니다.

## 실전에서 얻은 교훈

이 블로그의 `style.css`를 쓸 때는 대부분 클래스 선택자만 사용하고 id 선택자는 거의 쓰지 않았습니다. 그래야 나중에 스타일을 덮어쓸 때 우선순위 싸움을 하지 않아도 되기 때문입니다. `!important`는 최후의 수단으로만 남겨두는 게 좋다는 것도 다시 느꼈습니다.

## 정리

- 선택자는 구체적일수록 우선순위가 높다.
- id > 클래스 > 태그 순으로 점수가 매겨진다.
- 실무에서는 클래스 선택자 위주로 작성하면 우선순위 충돌을 줄일 수 있다.
