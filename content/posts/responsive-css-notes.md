# 반응형 CSS 정리 노트

작은 화면에서도 읽기 편한 사이트를 만들기 위해 정리한 노트입니다.

## 기본 원칙

본문 컬럼의 너비를 제한하고, 여백과 줄 간격을 넉넉히 주는 것이 가독성의 기본입니다. 픽셀 대신 `rem`, `%`, `ch` 같은 상대 단위를 쓰면 화면 크기에 따라 자연스럽게 늘어나고 줄어듭니다.

## 이미지와 코드블록

이미지는 항상 컨테이너 폭을 넘지 않아야 합니다.

```
img { max-width: 100%; height: auto; }
```

코드블록은 폭이 넓은 코드가 있을 때 페이지 전체를 깨뜨리지 않도록 자체적으로 가로 스크롤을 갖는 것이 좋습니다.

```
pre { overflow-x: auto; }
```

## 브레이크포인트

- 큰 사이트는 여러 단계의 브레이크포인트가 필요할 수 있습니다.
- 하지만 글 위주의 블로그라면 대부분 하나의 브레이크포인트로 충분합니다.
- 헤더처럼 요소가 몇 개 없는 영역은 `flex-wrap`만으로도 자연스럽게 줄바꿈됩니다.

마지막으로, 뷰포트 메타 태그를 잊지 마세요. 이게 없으면 모바일 브라우저가 페이지를 데스크톱 폭으로 렌더링한 뒤 축소해버립니다.
