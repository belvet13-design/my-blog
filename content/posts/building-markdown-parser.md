# 직접 만든 마크다운 파서 살펴보기

이 블로그는 marked.js 같은 라이브러리를 쓰지 않고, 마크다운을 HTML로 바꾸는 파서를 직접 만들었습니다. 오늘은 그 구조를 정리해봅니다.

## 왜 두 단계로 나눴을까

파서는 **블록 분리**와 **인라인 서식** 두 단계로 나뉩니다. 한 번에 정규식 하나로 모든 걸 처리하려고 하면 코드가 금방 엉키기 때문에, 먼저 줄 단위로 블록(제목, 목록, 코드블록 등)을 나누고, 그다음 각 블록의 텍스트 안에서 굵게/기울임/링크 같은 인라인 서식만 따로 처리합니다.

## 블록 분리

줄을 순회하면서 각 줄이 어떤 블록인지 분류합니다.

```
if (line.match(/^```/)) {
  // 코드블록: 닫는 ```까지 그대로 수집
}
if (line.match(/^#{1,6}\s+/)) {
  // 헤딩
}
if (line.match(/^[-*+]\s+/)) {
  // 비순서 목록
}
```

코드블록은 특별 취급이 필요합니다. 안의 내용은 인라인 서식을 적용하면 안 되고, HTML 이스케이프만 해서 그대로 `<pre><code>`에 넣어야 합니다.

## 인라인 서식은 순서가 중요하다

`parseInline` 함수는 정규식을 순서대로 적용합니다.

1. HTML 특수문자 이스케이프 (보안을 위해 가장 먼저)
2. 인라인 코드 `` `code` ``
3. 이미지 `![alt](url)`
4. 링크 `[text](url)`
5. 굵게 `**text**`
6. 기울임 `*text*`

이미지를 링크보다 먼저 처리하는 이유는 문법이 `!`만 다르고 나머지는 같아서, 링크 규칙이 먼저 실행되면 이미지 문법을 망가뜨리기 때문입니다. 굵게를 기울임보다 먼저 처리하는 것도 같은 이유입니다.

## 배운 점

표나 중첩 목록처럼 복잡한 문법은 과감히 지원 범위에서 뺐습니다. 개인 블로그에 필요한 문법만 지원하기로 하니, 코드량이 훨씬 적고 이해하기도 쉬웠습니다.
