# my-blog

마크다운 파일을 읽어서 블로그 웹사이트로 변환하는 정적 사이트 프로젝트.

## 기술 스택

- **프레임워크 없음.** 순수 HTML, CSS, JavaScript만 사용한다 (빌드 도구, 번들러, npm 패키지 없이).
- 마크다운 파싱은 외부 라이브러리 없이 직접 구현하거나, 부득이한 경우에만 가벼운 단일 파일 라이브러리를 CDN 없이 로컬에 벤더링해서 사용한다.
- 정적 파일이므로 서버 사이드 로직 없이 브라우저에서 직접 열거나 간단한 정적 호스팅으로 서비스 가능해야 한다.

## 디자인 원칙

- **깔끔하고 읽기 좋은 타이포그래피** 우선. 본문 가독성(줄 간격, 폭 제한, 폰트 크기)을 최우선으로 고려한다.
- **다크 모드 지원 필수.** `prefers-color-scheme` 미디어 쿼리를 기본으로 하고, 사용자가 수동으로 토글할 수 있는 스위치도 제공한다.
- **모바일 반응형 필수.** 좁은 화면에서도 여백, 폰트 크기, 네비게이션이 자연스럽게 동작해야 한다.
- 불필요한 장식이나 애니메이션은 지양하고, 콘텐츠(글) 자체가 돋보이도록 한다.

## 작업 시 유의사항

- 새 기능을 추가할 때 프레임워크나 패키지 매니저를 도입하려 하지 말 것 — 항상 순수 HTML/CSS/JS로 구현한다.
- CSS는 커스텀 속성(CSS variables)으로 라이트/다크 테마 색상을 관리한다.
- 마크다운 파일은 별도 콘텐츠 디렉터리에 두고, JS에서 fetch하여 파싱 후 렌더링하는 구조를 기본으로 한다.
