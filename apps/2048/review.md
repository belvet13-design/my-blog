# 2048 게임 — Review 결과

Build 서브에이전트가 구현한 `/apps/2048/`을 독립적으로 검증했다. 프리뷰 도구로 실제 브라우저(정적 서버, `http://localhost:8000/apps/2048/index.html`)에서 조작했고, 콘솔 로그를 확인했으며, 코드를 읽고 `preview_eval`로 핵심 로직에 임시 디버그 훅(`window.__TEST_HOOKS__`)을 걸어 다양한 보드 상태를 강제로 주입해 엣지 케이스를 검증했다. 검증이 끝난 뒤 디버그 훅은 완전히 제거했다.

## 발견한 문제

### 1. (심각, 수정함) 상/하 방향키가 서로 뒤바뀌어 있었음

`game.js`의 `DIRECTION_ROTATIONS = { left: 0, up: 1, right: 2, down: 3 }`에서 `rotateBoard`(90도 시계방향 회전)를 이용해 4방향을 "왼쪽 이동"으로 통일하는 과정의 회전 값이 up/down에 대해 반대로 매핑되어 있었다.

**증상**: 단일 타일 `(1,1)`을 놓고 각 방향으로 이동시키는 테스트에서
- `move('up')` → 타일이 `(3,1)`(맨 아래줄)로 이동 (기대: `(0,1)`)
- `move('down')` → 타일이 `(0,1)`(맨 윗줄)로 이동 (기대: `(3,1)`)

`left`/`right`(rot=0, rot=2)는 정상이었고, `up`/`down`(rot=1, rot=3)만 반대로 동작했다. 실제 키보드로 위/아래 화살표를 누르면 타일이 반대 방향으로 움직이는 치명적인 버그로, 게임 플레이가 불가능한 수준이었다.

**수정**: `DIRECTION_ROTATIONS`를 `{ left: 0, up: 3, right: 2, down: 1 }`로 변경(`apps/2048/game.js` 11번째 줄). 수정 후 4방향 모두 단일 타일 이동 테스트와 컬럼 병합 테스트(`up`/`down` 각각 병합 점수까지) 재검증해서 정상 동작 확인.

### 2. (사소, 기록만 함) 승리/게임오버 오버레이가 뜰 때 스크린샷상 타일이 안 덮인 것처럼 보임

처음 스크린샷에서 오버레이가 보드 일부만 덮는 것처럼 보여 의심했으나, `getBoundingClientRect()`로 `#board`, `#board-container`, `#game-message`의 실제 좌표/크기를 측정한 결과 셋 다 정확히 동일한 사각형(420×420, 동일 좌표)을 차지하고 있어 CSS(`position:absolute; inset:0;`)가 의도대로 보드 전체를 덮고 있음을 확인했다. 단순히 반투명 배경(`rgba(.., 0.85)`) 위에 밝은 타일 색상이 겹쳐 보이는 시각적 착시였다(게임오버 체크메이트 시나리오 스크린샷에서는 모든 타일이 고르게 어둡게 눌린 것을 재확인함). 실제 버그 아님, 코드 수정 없음.

### 3. (사소, 기록만 함) `render()`가 매번 `.tile` 전체를 재생성

spec 4.4의 "최소 구현 범위"에서 명시적으로 개별 타일 슬라이드 애니메이션(타일 id 추적)은 nice-to-have로 남겨도 된다고 했으므로, 현재 구현(매 렌더마다 `.tile` 전부 제거 후 재생성)은 spec 범위 내에서 문제 없음. 다만 타일이 부드럽게 미끄러지는 시각 효과는 없고 즉시 나타남/사라짐만 있다(신규/병합 타일에는 `tile-new`/`tile-merged` 애니메이션 적용됨, 확인함). 스타일 취향 차이라 수정하지 않음.

### 4. (사소, 기록만 함) `mergedMask`는 spec에 없는 추가 기능

Build 서브에이전트가 spec의 pseudo-code에는 없던 `mergedIndices`/`mergedMask` 추적을 추가해 병합된 타일에 정확히 `tile-merged` 애니메이션 클래스를 부여하고 있다. spec을 벗어나지 않는 개선이며 실제로 병합 시 애니메이션이 올바른 타일에만 적용되는 것을 확인했다. 문제 아님.

## 최종 결론

**수정 전에는 상/하 이동이 반대로 동작하는 치명적 버그로 인해 배포 불가 상태**였다. `DIRECTION_ROTATIONS`의 up/down 값을 수정한 뒤 슬라이드+병합, 4방향 회전, 게임 오버 판정, 승리 판정, 점수/최고점수, 다크모드, 반응형/스와이프, 뒤로가기 링크를 모두 재검증했고 전부 spec대로 동작하며 콘솔 에러도 없다.

**결론: 수정 완료, 배포 가능.**

## 실제 수정한 내용

- `apps/2048/game.js` 11번째 줄: `var DIRECTION_ROTATIONS = { left: 0, up: 1, right: 2, down: 3 };` → `var DIRECTION_ROTATIONS = { left: 0, up: 3, right: 2, down: 1 };` (상/하 방향키가 반대로 동작하던 버그 수정)
- 검증 과정에서 임시로 추가했던 `window.__TEST_HOOKS__` 디버그 노출 코드는 검증 완료 후 제거함(최종 `game.js`에는 남아있지 않음).
- `index.html`, `style.css`는 수정하지 않음.
