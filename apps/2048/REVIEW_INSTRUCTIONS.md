# Review 지침 — 2048 게임 검증

## 역할

너는 Review 서브에이전트다. Build 서브에이전트가 이미 구현한 `/apps/2048/` 게임을 독립적으로 검증하는 것이 임무다. 너는 구현자가 아니라 검수자다. 구현 의도를 신뢰하지 말고 실제로 브라우저에서 동작시켜보고, 코드를 읽어서 확인해라.

## 범위

- 검증 대상: `C:\Users\User\Desktop\my-blog\apps\2048\index.html`, `style.css`, `game.js`
- 참고 문서: `C:\Users\User\Desktop\my-blog\apps\2048\spec.md` (설계 기준), 8장에 Review 체크리스트가 이미 있으니 그대로 활용해라.
- 결과물: `C:\Users\User\Desktop\my-blog\apps\2048\review.md` 작성.
- 문제를 발견하면 `index.html`, `style.css`, `game.js` 세 파일 안에서 직접 수정해도 된다.
- 이 세 파일과 `review.md` 외에는 어떤 파일도 수정하지 마라. 특히 블로그 루트의 `index.html`, `css/`, `js/`, `content/`, 그리고 `spec.md`, `BUILD_INSTRUCTIONS.md`는 절대 건드리지 않는다.

## 검증 방법

1. 프리뷰 도구(정적 서버)로 `/apps/2048/index.html`을 실제 브라우저에서 열어라. Bash로 서버를 띄우지 말고 preview 계열 도구를 사용해라.
2. spec.md 8장 체크리스트를 하나씩 실제로 조작하며 확인해라:
   - 방향키 4방향 이동/병합
   - 연속 3개 이상 병합 시 1회만 병합되는지 (`[2,2,2,0] → [4,2,0,0]`)
   - 변화 없는 이동 시 새 타일 미생성
   - 게임 오버 판정 정확성
   - 2048 도달 시 승리 메시지 + 계속하기
   - 최고 점수 localStorage 유지(새로고침 후에도)
   - 모바일 뷰포트(375px)에서 레이아웃과 스와이프
   - 다크모드 토글
   - `../../index.html` 뒤로가기 링크가 실제로 블로그 루트로 연결되는지
3. 브라우저 콘솔에 에러/경고가 없는지 확인해라.
4. 코드를 읽고 spec.md의 핵심 로직(슬라이드+병합, 4방향 회전 통일, 게임 오버 판정)이 pseudo-code와 동일한 결과를 내는지 검토해라. 필요하면 `preview_eval`로 특정 보드 상태를 강제로 넣어 엣지 케이스를 테스트해라.
5. 발견한 문제는 심각도와 상관없이 review.md에 전부 기록하되, **동작을 깨뜨리는 버그는 네가 직접 고쳐라.** 사소한 스타일/취향 차이는 기록만 하고 고치지 않아도 된다.

## review.md 형식

- 검증한 항목과 결과(통과/실패)를 체크리스트 형태로 정리
- 발견한 문제와 그 문제를 어떻게 고쳤는지(또는 왜 안 고쳤는지)
- 최종 결론: 배포 가능 여부

작업이 끝나면 review.md의 핵심 결론과 실제로 수정한 내용이 있다면 무엇인지 보고해라.
